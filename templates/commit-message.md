# コミットメッセージ テンプレート
# （Conventional Commits 形式）

---

## 📝 基本フォーマット

```
<種別>(<スコープ>): <タイトル（何をしたか）>

<本文（なぜそうしたか、何が変わったか）>

<フッター（関連IssueやBreaking Changeなど）>
```

---

## 🏷️ 種別（type）一覧

| 種別 | 使う場面 | 例 |
|------|---------|-----|
| `feat` | 新機能の追加 | `feat: ログイン画面に新規登録リンクを追加` |
| `fix` | バグの修正 | `fix: カート数量が正しく更新されない問題を修正` |
| `docs` | ドキュメントの変更 | `docs: README に環境構築手順を追記` |
| `style` | コードのフォーマット（動作変更なし） | `style: インデントをスペース2つに統一` |
| `refactor` | リファクタリング（機能変更なし） | `refactor: UserProfile を3つのコンポーネントに分割` |
| `test` | テストの追加・修正 | `test: calculateDiscount のユニットテストを追加` |
| `chore` | ビルドや補助ツールの変更 | `chore: node_modules を更新` |
| `perf` | パフォーマンス改善 | `perf: 商品一覧の画像を遅延読み込みに変更` |

---

## ✅ 良いコミットメッセージの例

```
feat(auth): ログイン画面にパスワードリセット機能を追加

- パスワードリセットリンクをログインフォームの下に追加
- メールアドレス入力画面（/reset-password）を新規作成
- Tailwind CSS の既存クラスを使いデザインを統一

Closes #42
```

---

## ❌ 悪いコミットメッセージの例

```
修正した          ← 何を修正したのか分からない
fix               ← タイトルがない
いろいろ変えた    ← 曖昧すぎる
WIP               ← 完成していないものをコミットしない（基本的に）
```

---

## 🤖 Claude Code に生成させる場合のプロンプト

```
現在 `git add` されている変更内容を読み取り、
Conventional Commits の形式（feat:, fix:, docs: など）で
適切なコミットメッセージを作成して `git commit` を実行してください。
タイトルと、変更の理由を含めた詳細な説明を日本語で記述してください。
```
