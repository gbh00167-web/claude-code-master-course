## 💳 Claude Code の料金のしくみ

Claude Code は **従量課金制（使った分だけ払う）** です。
Web 版の Claude Pro（月額固定）とは**全く別の契約**です。

### プリペイド方式
Suica のようにあらかじめチャージしておく方式です。
**最初は $5〜10（約750〜1500円）** のチャージから始めましょう。

### 月間上限額の設定（必須！）
```
Anthropic Console → Settings → Limits → Monthly Spend Limit
```
**$10〜20** に設定しておけば、万が一 AI がバグで無限ループに入っても安心です。

## 📊 トークンとは何か

AI が文章を処理する最小単位。簡単に言うと「文字数のようなもの」です。

| 目安 | トークン数 |
|------|-----------|
| 英語 1単語 | 約 1トークン |
| 日本語 1文字 | 約 1〜2トークン |
| コード 1行 | 約 10〜30トークン |
| ファイル 1本（小〜中規模） | 約 500〜3,000トークン |

`in`（AIに送る量）と `out`（AIが返す量）は別々に課金されます。
一般的に `out` の方が単価が高いです。

## ⚠️ コストが膨らむ3大原因

### 原因1：会話が長くなりすぎる（雪だるま式増大）
LLM は「過去の会話全部」を毎回送信します。
→ **対策：10〜15往復したら `/compact` を実行**

```bash
# コスト確認
/stats

# 圧縮（要約）して節約
/compact
```

### 原因2：node_modules を読み込んでしまう
数万ファイルあるため、一瞬でトークンが爆発します。
→ **対策：`.claudeignore` に `node_modules/` を必ず登録**

```bash
# .claudeignore の確認
cat .claudeignore
```

### 原因3：丸投げして大量のやり直しが発生する
「全部作って」→ 的外れな実装 → やり直し → トークン2倍消費
→ **対策：段階的に小さく指示する（Incremental Prompting）**

## 💡 賢く節約する3つの戦術

### 戦術1：タスクの区切りでセッションをリセット

```bash
# 一つの機能が完成したら
/clear  # または /exit して再起動

# 別のタスクを新鮮なコンテキストで開始
claude
```

### 戦術2：読み取り専用モードを活用

```bash
# コードを変更せずに質問するだけなら /ask を使う
# → 不要なファイル書き換えがなくなる = 無駄なトークン消費なし
/ask この関数の処理を説明して
```

### 戦術3：プロジェクトルールを CLAUDE.md に書いておく

毎回プロンプトに「TypeScript を使って」と書かなくて済む分、トークンが節約できます。

## 📈 コスト感覚を養う習慣

| タイミング | アクション |
|-----------|-----------|
| 大きな機能実装の前後 | `/stats` でコストを確認 |
| 10〜15往復ごと | `/compact` で圧縮 |
| 1つのタスク完了時 | `/clear` または `/exit` でリセット |
| 毎週 | Anthropic Console でログを確認 |

## 🆘 「API 破産」の緊急対処法

もし想定外のコストが発生した場合：

1. **即座に `/exit`** で Claude Code を終了
2. **Anthropic Console** にアクセスしてセッションを確認
3. **Spend Limits** でそれ以上の請求が出ないよう上限を下げる
4. 原因を特定して（大量のファイル読み込み等）`.claudeignore` を修正
