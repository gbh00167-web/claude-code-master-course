# プロンプト早見表 — 場面別コピーして使えるプロンプト集

> `〇〇` の部分をあなたのケースに書き換えて使ってください。

---

## フォルダ・ファイルの把握

### フォルダの内容を確認する
```
このフォルダに何がありますか？
ファイルの一覧と、それぞれがどんなファイルか一言で教えてください。
```

### 特定のファイルを探す
```
〇〇（キーワード）という名前のファイルをフォルダ内で探して、
見つかったファイルのパスと概要を教えてください。
```

---

## ファイルを読んで成果物を作る

### 議事録の要約
```
〇〇.txt を読んで、以下の形式でまとめてください：

## 決定事項
（箇条書き）

## アクションリスト
担当者名：タスク（期限）の形式で

## 次回予定
```

### 複数ファイルの横断集計
```
〇〇/ フォルダにある全ての .txt ファイルを読んで、
〇〇（集計したい内容）を一覧にまとめてください。
結果は 〇〇.md として保存してください。
```

### 長い資料の要点だけ抽出
```
〇〇.txt を読んで、以下の点だけ教えてください：
- 〇〇（知りたい項目1）
- 〇〇（知りたい項目2）
- 〇〇（知りたい項目3）
全文の要約は不要です。上記の点だけ答えてください。
```

## 文書を作る・編集する

### 新しい文書を作る
```
以下の条件で〇〇（文書の種類）を作って、〇〇.txt として保存してください：
- 〇〇（条件1）
- 〇〇（条件2）
- 文体：ですます調
- 分量：〇〇字程度
```

### 既存文書の一部を書き直す
```
〇〇.txt を読んで、〇〇（書き直したい箇所）を
〇〇（どう直したいか）に書き直してください。
それ以外の部分は変えないこと。
```

### テンプレートから文書を量産する
```
〇〇-template.txt を参考に、以下の内容で新しいファイルを作ってください：
- 〇〇：〇〇
- 〇〇：〇〇
ファイル名は 〇〇-YYYYMMDD.txt 形式で保存してください。
```

## CLAUDE.md の作成・更新

### CLAUDE.md を作る
```
このフォルダに CLAUDE.md を作ってください。
内容：
- 私の職種：〇〇
- 文体：ですます調
- よく使う情報：〇〇
```

### CLAUDE.md に追記する
```
CLAUDE.md に以下を追加してください：
「〇〇（追加したいルール）」
次回から毎回言わなくて済むようにしたい。
```

## AIが間違えたときの対応

### 部分的に修正させる
```
作ってもらった内容ですが、〇〇の部分が意図と違います。
〇〇のように変更してください。
それ以外は変えないこと。
```

### 最初からやり直させる
```
一旦リセットしてやり直しましょう。
今度は〇〇という制約を守って作ってください。
```

### なぜその出力になったか確認する
```
なぜ〇〇という結果になったか教えてください。
私の指示のどこが曖昧でしたか？
```
