## 🔍 第3章：コード探索

### プロジェクト全体把握

```
新しくこのプロジェクトに参画したエンジニアとして、
ディレクトリ構造を要約し、採用されている技術スタックと

```

### 特定の関数を追跡する
```

その関数がどのファイル・コンポーネントから呼び出されているか、
影響範囲をすべてリストアップしてください。
```

### データフローを解説させる
```
/ask ユーザーが `〇〇ページ` で `△△ボタン` を押してから、
`□□処理` が完了するまでのデータの流れを
ステップバイステップで分かりやすく解説してください。
```

## ⚙️ 第4章：実装

### 小さな変更（制約付き）
```
`src/components/〇〇.tsx` を編集してください。
〇〇の部分に△△の機能を追加します。

【制約事項】
- 新しい npm パッケージはインストールしないでください
- 既存の Tailwind CSS クラスを使ってスタイリングしてください
- ロジックの変更範囲はこのファイルのみにしてください
```

### 計画立案（実装の前に）
```
〇〇機能を追加したい。
実際のコードを書く前に、まず以下の2点だけ教えてください：
1. 新規作成・編集が必要なファイルのリスト
2. どのようなコンポーネント構成にするかの実装計画

いきなりファイルの編集は行わないでください。
```

### 大規模リファクタリング
```
`src/components/〇〇.tsx` が肥大化して保守性が低くなっています。
アプリケーションの動作と見た目は一切変更せずに、
以下のコンポーネントに分割してください：
1. `〇〇Header.tsx`（〇〇の部分）
2. `〇〇Body.tsx`（△△の部分）
分割に伴う import 文のパス修正も漏れなく行ってください。
```

### Linter エラーを一括解消
```
`npm run lint` と `npm run type-check` を自分で実行して
エラーログを読み取り、プロジェクト全体のエラーがゼロになるまで
必要なファイルをすべて修正してください。
`any` 型は処理内容から適切な型に推論して定義してください。
```

## 🐛 第5章：デバッグ・テスト

### ビルドエラーを自動修正
```
`npm run build` を実行して、エラーが出たらログを読んで
原因を特定し、必要なファイルをすべて修正してください。
エラーが完全になくなるまで修正と再ビルドを繰り返してください。
```

### 論理バグの調査
```
〇〇画面で問題が発生しています。
「△△ボタンを押しても□□が起きない」という症状です。
まず関連するコンポーネントに `console.log` を適切に仕込んで
処理の流れを可視化してください。
原因を特定して修正したら、仕込んだ `console.log` を削除してください。
```

### テストコードの自動生成
```
`src/utils/〇〇.ts` にある関数のユニットテストを作成してください。
テストファイルは `src/utils/〇〇.test.ts` として新規作成します。
正常系だけでなく、異常系とエッジケース（マイナス値、空文字、nullなど）も
徹底的に網羅してください。テストフレームワークは Vitest を使用してください。
```

### テスト自動ループ
```
`npm run test` を実行してください。
FAIL するテストがあれば：
1. テストコードが間違っているのか、本体ロジックが間違っているのかを分析する
2. すべてのテストが PASS するようにファイルを修正する
3. 修正後、再度テストを実行して確認する
全テストが緑色（PASS）になるまでこのループを続けてください。
```
