# コマンドリファレンス 早見表

> 印刷して手元に置いておくと便利です

---

## 起動・終了

| コマンド | 何をするか |
|---------|-----------|
| `claude` | Claude Code を起動する |
| `claude --version` | バージョンを確認する |
| `/exit` | 終了してターミナルに戻る |
| `Ctrl + C` | 強制終了 |

---

## 会話中に使えるコマンド

| コマンド | 何をするか | いつ使う |
|---------|-----------|---------|
| `/cost` | 現在のコストを表示 | 使用料が気になったとき |
| `/clear` | 会話履歴をリセット | 全く別の作業に切り替えるとき |
| `/exit` | Claude Code を終了 | 作業終了時 |

---

## 作業の止め方

| 操作 | 効果 |
|------|------|
| `Esc` キー | 実行中の作業を停止。入力待ちに戻る |
| `Ctrl + C` | 強制終了 |

## 「Y/n」の判断基準

Claudeがファイルを変更しようとすると確認画面が出ます：

| Claudeがしようとしていること | 判断 |
|---------------------------|------|
| ファイルを読む | 基本的に許可してOK |
| 新しいファイルを作成する | 内容を確認してから許可 |
| 既存ファイルを書き換える | **必ず内容を確認してから**許可 |
| ファイルを削除する | 慎重に確認。不安なら「n」で拒否して理由を聞く |

## Git 安全ネット コマンド

```bash
# 作業前のセーブ（重要なファイルを変更する前に必ず実行）
git add . && git commit -m "作業前のセーブ"

# AIが変更した内容を確認する
git diff

# 全ての変更を元に戻す（最後のセーブ時点に戻る）
git restore .

# 特定のファイルだけ元に戻す
git restore ファイル名.txt
```

## よく使うプロンプトパターン

### フォルダの内容を把握する
```
このフォルダに何がありますか？ファイルの一覧と、それぞれの概要を教えてください。
```

### ファイルを読んで要約する
```
〇〇.txt を読んで、要点を3つ箇条書きでまとめてください。
```

### 新しい文書を作る
```
以下の条件で〇〇の下書きを作って、〇〇.txt として保存してください：
・[条件1]
・[条件2]
```

### 既存ファイルを修正する
```
〇〇.txt の〇〇の部分を〇〇に書き直してください。
それ以外の部分は変えないこと。
```

### AIが間違えたとき
```
今作ってもらった内容ですが、〇〇の部分が意図と違います。
〇〇のように変更してください。
```
