# Claude Code コマンドリファレンス 早見表

> 印刷して手元に置いておくとすぐ使えます

---

## 🖥️ 起動・終了

| コマンド | 何をするか |
|---------|-----------|
| `claude` | Claude Code を起動（プロジェクトルートで実行） |
| `claude --version` | バージョン確認 |
| `/exit` | Claude Code を終了してターミナルに戻る |
| `Ctrl + C` × 2回 | 強制終了 |

---

## 💬 対話中のスラッシュコマンド

| コマンド | 何をするか | いつ使う |
|---------|-----------|---------|
| `/ask <質問>` | **読み取り専用**モードで質問（ファイルを変更しない） | コードを読んで理解したい時 |
| `/compact` | 会話履歴を要約して圧縮（コスト節約） | やり取りが10往復を超えた頃 |
| `/clear` | 会話履歴を完全リセット（記憶ゼロに） | 全く別のタスクに切り替える時 |
| `/stats` | 現在のトークン消費量とコストを表示 | コストが気になった時 |
| `/exit` | Claude Code を終了 | 作業終了時 |

## 🔑 Allow / Refuse の判断基準

| Claudeが求めてきた操作 | 判断 | 理由 |
|----------------------|------|------|
| `ls`, `cat`, `find`（読み取り系） | ✅ 許可（A） | ファイルを壊さないから |
| `npm install <パッケージ名>` | ✅ 確認してから許可（y） | 意図したものか確認 |
| `npm run test`, `npm run build` | ✅ 許可（y） | 開発の通常操作 |
| `git commit`, `git push` | ⚠️ **必ず内容確認してから**（y） | 何が含まれているか確認必須 |
| `rm <ファイル>` | ⚠️ 慎重に確認（n → 理由を聞く） | 削除は取り戻せない場合がある |
| 見覚えのないシェルスクリプト実行 | ❌ 拒否（n） | 意図不明なコマンドは危険 |

## 🛡️ Git 安全ネット コマンド集

```bash
# 作業前のセーブデータ作成（必ず実行する習慣を）
git add .
git commit -m "作業前のセーブ"

# AIが変更した差分を確認する（コミット前に必ず実行）
git diff

# AIが書いたコードを全部元に戻す（リセット）
git restore .

# 特定のファイルだけ元に戻す
git restore src/components/SomeFile.tsx

# コミット履歴を確認
git log --oneline
```

## 💡 よく使うプロンプトパターン

### 全体把握
```
このプロジェクトの概要と主要なディレクトリ構成、アーキテクチャの特徴を教えて。
```

### 計画先行（実装前の確認）
```
〇〇機能を追加したい。コードを書く前に、修正が必要なファイルのリストと実装手順の計画を教えて。実装は計画を確認してから始めること。
```

### 安全なコード調査
```
/ask `loginUser` という関数がどこで定義されているか探し出し、呼び出し元をすべてリストアップして。
```

### エラー自動修正
```
`npm run build` を実行して、エラーが出たらログを読んで原因を特定し、エラーがゼロになるまで修正を繰り返して。
```

### コミットの自動化
```
現在 `git add` されている変更内容を読み取り、Conventional Commits 形式（feat:, fix: など）で適切なコミットメッセージを作成して `git commit` を実行して。
```
