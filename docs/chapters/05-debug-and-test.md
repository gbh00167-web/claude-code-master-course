# 第5章：デバッグとテスト — ターミナルの出力を武器にする

> **学習目的：** エラーログをAIに渡し、バグ特定から修正、テスト生成までを自動化する
> **学習成果：** 赤いエラーログへの恐怖心を払拭し、デバッグ時間を劇的に削減できる
> **目安時間：** 60分

## 5.1 自動デバッグ：エラーログの究極の解消法

AI駆動開発においてエラーが出た時の最適な行動は、**「エラーが出たという事実を、そのままClaudeに丸投げすること」**です。

### ターミナルコマンドをAIに実行させる

❌ **疲れる旧来のデバッグ方法**
```
人間がターミナルで npm run build を打つ
→ エラーが出る
→ エラー文をコピーする
→ ブラウザを開いてAIに貼り付ける
```

⭕️ **賢い自動デバッグのプロンプト**
```
> `npm run build` を実行して、エラーが出たらログを読んで原因を特定し、
  必要なファイルをすべて修正してください。
  エラーが完全になくなるまで修正と再ビルドを繰り返してください。
```

Claudeが自律的に行うこと：
1. `npm run build` を実行
2. エラーログを読んで原因を特定
3. ファイルを修正
4. 再度 `npm run build` を実行して確認
5. 全エラーが消えたら報告

### 論理バグ（画面は表示されるが動きがおかしい）の調査

```
> お問い合わせフォームで問題が起きています。
  「送信」ボタンを押しても何も反応しません。
  まず関連するコンポーネントに `console.log` を仕込んで処理を可視化してください。
  原因を特定して修正したら、仕込んだ `console.log` を削除してください。
```

## 5.2 テスト生成：品質の担保

「動いた！よかった！」で終わってはいけません。**同じバグが再発しないようにテストコードで保護する**のがプロの仕事です。

### ユニットテストの自動生成

```
> `src/utils/calculateDiscount.ts` にある割引計算ロジックのユニットテストを作成してください。
  テストファイルは `src/utils/calculateDiscount.test.ts` として新規作成します。
  正常系だけでなく、異常系やエッジケース（マイナスの金額、0円、小数点など）も
  徹底的に網羅してください。テストフレームワークはVitestを使用してください。
```

### テスト自動ループ

```
> `npm run test` を実行してください。
  FAILするテストがあれば以下で対処してください：
  1. テストコードが間違っているのか、本体ロジックが間違っているのかを分析する
  2. すべてのテストがPASSするようにファイルを修正する
  3. 修正後、再度テストを実行して確認する
  全テストが緑色（PASS）になるまでこのループを続けてください。
```

> 💡 **「赤いエラー文字」はもう恐怖の対象ではなく、AIをより賢く動かす「美味しいエサ」です。**

## 5.3 環境操作：シェルコマンドとGit操作の融合

### 複雑なコマンドの実行代行

```
> 現在のディレクトリとサブディレクトリにある、すべての `.js` ファイルを探し出し、
  ファイル名を `.ts` に変更するコマンドを実行してください。
  実行前に「どのファイルが変更されるか」のリストを提示して確認を取ってください。
```

### Gitコンフリクトの解消

```
> 現在 `git merge` でコンフリクトが発生しています。
  コンフリクトが起きているファイルを調査し、両方の変更内容の意図を汲み取って、
  適切にマージするようにファイルを修正してください。
```
