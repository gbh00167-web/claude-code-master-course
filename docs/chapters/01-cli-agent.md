## 1.1 概念：なぜ今「CLIエージェント」なのか

### チャット型AI（受動的）vs CLIエージェント（能動的）

Webブラウザで使うChatGPTやWeb版のClaudeは、本質的に**「受動的なAI」**です。あなたが質問（プロンプト）を入力し、それにAIが回答を生成して終わる「一問一答」のモデルです。AIが自らあなたのパソコンの中を覗き込んだり、勝手にファイルを書き換えたりすることはできません。

一方で、Claude CodeのようなCLIエージェントは**「能動的なAI」**です。

あなたが「この機能を作って」と自然言語で指示を出すと、AIは自らターミナル上で行動計画を立て、動き出します。プロジェクト内を検索し、対象ファイルを見つけて読み込み、コードを書き換え、さらにテストコマンドを実行してエラーが出ないか自己評価する、という一連のサイクルを自動で回してくれます。

| 種別 | 例え | 役割 |
|------|------|------|
| チャット型AI | 物知りで優秀なアドバイザー | 質問に答えてコードの書き方を教えてくれる |
| CLIエージェント | 自律型のペアプログラマー | 自ら現状を観察し、計画を立てて、実装からテストまで手を動かしてくれる |

> 💡 **初心者の疑問：AIが勝手に動くって、大事なファイルを消されたりしないか怖いです。**
> Claude Codeは「何かファイルを変更する前」に必ず人間に「これを実行してもよいですか？ (Allow/Refuse)」と許可を求めてくる設定が組み込まれています。また、Gitと組み合わせることで、万が一間違った修正をされても一瞬で元に戻せる「安全ネット」を用意できます。

### コンテキスト共有の圧倒的利点

CLIエージェント最大の武器は、**「AIが自らコンテキスト（文脈・現状）を取りに行ける」**という点です。あなたが何も説明しなくても、Claude Codeはターミナルを通じて `package.json` や `tsconfig.json` といった設定ファイルを読み込んで技術スタックを把握します。

料理に例えるなら、レシピを聞くために、自宅の冷蔵庫の中身をすべてスマホでテキスト入力して送信していたような作業が不要になるのです。

## 1.2 準備：動作環境のセットアップ

### 必須環境の確認

**1. Node.js（バージョン18以上）**

```bash
# バージョン確認
node -v
# v18.17.0 のように表示されればOK
```

入っていない場合は [nodejs.org](https://nodejs.org/) から安定版（LTS版）をインストールしてください。

**2. Git（実質的な必須要件）**

Gitはコードの履歴を保存するシステムです。Claude Codeを使う上で、Gitは**「最強の安全ネット（フェイルセーフ）」**になります。

AIが「良かれと思って」大量のファイルを変更してしまっても、以下のコマンドで一瞬にして元に戻せます。

```bash
# AIが加えた変更をすべて取り消す（魔法のコマンド）
git restore .
```

> ⚠️ **鉄則：作業前に必ずコミットしてセーブデータを作る**
> ```bash
> git add .
> git commit -m "作業前のセーブ"
> ```

### APIの支払い設定と「API破産」の防ぎ方

> 🚨 **重要：Claude Codeの利用には、Anthropic APIの利用料金（従量課金）がかかります。**
> これはWeb版の「Claude Pro（月額固定）」とは**全く別の契約**です。

1. **Anthropic Console にアクセス：** [console.anthropic.com](https://console.anthropic.com/) でアカウント作成・ログイン
2. **支払い設定（Billing）：** クレジットカードを登録
3. **クレジットのチャージ：** まずは $5〜10（約750〜1500円）の少額をチャージ
4. **月間上限額の設定（必須！）：** `Settings` → `Limits` から `Monthly Spend Limit` を `$10〜20` に設定

### インストールコマンド

```bash
npm install -g @anthropic-ai/claude-code

# バージョン確認（成功すれば数字が表示される）
claude --version
```

> 💡 **`-g` って何ですか？**
> 「グローバル（Global）」の略です。これを付けることで、パソコンのどのフォルダに移動しても `claude` コマンドが使えるようになります。

## 1.3 実践：最初の起動と基本設定

### セッションの開始

プロジェクトのルートディレクトリに移動して、起動します。

```bash
cd your-project-folder
claude
```

**初回起動時：** ブラウザが自動的に立ち上がり、Anthropicアカウントでの認証を求められます。ログインして「Authorize（許可）」をクリックしてください。（昔のツールのようにAPIキーをコピペする面倒な作業は一切不要です）

### プロジェクトの初期認識

起動すると `>` というプロンプトが表示されます。ここからClaudeとの対話スタートです。

```
> このプロジェクトの概要と主要なディレクトリ構成、アーキテクチャの特徴を教えて。
```

Enterを押すと、画面に `Running ls` や `Reading package.json` といった文字が高速で表示されます。Claudeが自ら「行動」を起こしているのが分かります。

### コスト管理の必須習慣（/stats コマンド）

```
> /stats
```

今のセッションで消費したトークン数と推定コストが表示されます。

```
Tokens: 15,240 in / 1,200 out
Cost: $0.06
```

> 💡 **実務でのワンポイント：**
> 大きな作業を依頼した後はこまめに `/stats` を叩く癖をつけましょう。「この作業を任せると大体10円くらいかかる」という肌感覚が育ってきます。

セッションを終了するには：

```
/exit  または  Ctrl + C を2回
```

## まとめ

- CLIエージェントは「能動的なAI」——自らファイルを読み、コードを書き、テストまで行う
- Claude Codeはあなたのローカル環境に直接アクセスできるため、コンテキストの伝達が不要
- APIは従量課金制。月間上限額の設定と `/stats` でのコスト確認を習慣にする
- Gitの安全ネット（作業前コミット）は命綱——絶対に設定する
