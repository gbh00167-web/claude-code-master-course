## 3.1 Claude Code の「考えて動く」仕組み

Claude Code に仕事を頼むと、画面に次々と文字が表示されます。これはClaudeが「思考 → 行動 → 確認」のサイクルを繰り返しているプロセスです。

### 料理人の例えで理解する

Claude Code は、レシピを渡せば自分で冷蔵庫を開けて食材を確認し、調理して、完成品を出してくれる料理人のようなものです。

```
あなた：「議事録フォルダを読んで、要約レポートを作って」

Claude Code の内部動作：
  1. 議事録フォルダを探す（読む）
  2. 中のファイル一覧を確認する（読む）
  3. 各ファイルを開いて内容を把握する（読む）
  4. 要約をまとめる（考える）
  5. レポートファイルを作成する（書く ← ここで確認を求める）
  6. 完了を報告する
```

このサイクルを**エージェントループ**（エージェント＝自律的に動く存在、ループ＝繰り返し）と呼びます。

## 3.2 良い指示と悪い指示の違い

Claude Code への指示は「自然な日本語」で構いません。ただし、より良い結果を得るには、**「何を」「どのように」「どんな形で」** を含めると格段に精度が上がります。

### 指示の基本パターン

❌ **曖昧な指示**

```
> 議事録をなんとかして
```

→ 「なんとかする」の定義が不明。要約？整形？ファイル名変更？

✅ **具体的な指示**

```
> meeting-notes/ フォルダにある今週の議事録ファイルを読んで、
  次の形式でアクションリストを作って：
  - 担当者名：タスク内容（期限）
  の形式で箇条書き、ファイル名は action-list-今週の日付.md で保存
```

### 制約を加えると精度が上がる

```
> 提案書の下書きを作って。
  ただし：
  - 1500字以内に収めること
  - 専門用語は使わず平易な言葉で
  - 価格の話はまだ書かないこと
```

「ただし」以降の制約が、Claudeの出力品質を大きく引き上げます。

### 「段階的に確認しながら進む」指示

長い作業を一気に依頼するより、段階に区切って確認しながら進めると、方向がズレたときのリスクが小さくなります。

```
> まず議事録を読んで、どんな内容が含まれているか教えて。
  （内容を確認してから）
> わかった。では参加者のアクションリストだけ抜き出して。
  （確認してから）
> それをsummary.mdとして保存して。
```

## 3.3 作業を途中で止める方法

Claude Code が長い作業を始めたとき、「やっぱり方向が違う」「一旦止めたい」と感じることがあります。

### すぐに止める（Escape キー）

```
Esc キーを押す
```

現在実行中の作業を中断し、入力待ち状態に戻ります。ファイルへの変更は、中断時点まで反映されている場合があります。

### 緊急停止（Ctrl + C）

```
Ctrl + C
```

作業を即時強制終了します。ターミナルの制御が戻ります。

### 中断後の確認

中断した後は、どこまで作業が進んだかを確認しましょう：

```
> 今どこまで作業が完了していますか？完成していないものはありますか？
```

## 3.4 「確認モード」と「自動承認モード」

Claude Code には、作業の進め方に2つのモードがあります。

### デフォルト（確認しながら進む）

初期設定では、ファイルを変更するたびに確認を求めます。安全ですが、大量のファイルを処理するときは「Y」を押す回数が多くなります。

```
Claude wants to create: summary.md
Do you want to allow this? [Y/n]
```

### 自動承認モード（--dangerously-skip-permissions）

```bash
claude --dangerously-skip-permissions
```

このオプションを付けて起動すると、確認なしに作業が進みます。大量ファイルの処理に便利ですが、**意図しない変更も止まらずに実行される**ので注意が必要です。

> **初心者の方へ：** 最初のうちはデフォルトの「確認しながら」モードで使うことをお勧めします。Claude Code の動作に慣れてきたら、自動承認モードを試してみてください。

> **自動承認を使う前に：** 必ず `git commit` でセーブデータを作ること。万一問題が起きても `git restore .` で元に戻せます。

## 3.5 コストを抑えるためのコツ

Claude Code は使えば使うほどAPIの利用料金が発生します。無駄な消費を避けるためのポイントを押さえておきましょう。

### 現在のコストを確認する

```
/cost
```

セッション開始からの合計コストが表示されます。

### コストを抑えるポイント

| 状況 | 対処法 |
|------|--------|
| 同じ内容を何度も送っている | 指示をまとめて一度に送る |
| 大量のファイルを毎回読み直している | 必要なファイルを指定して絞り込む |
| 長い会話が続いている | `/clear` でセッションをリセット |
| 試行錯誤が多い | まず小さいファイルで試してから本番に適用 |

### セッションのリセット

```
/clear
```

会話履歴をリセットし、コンテキスト（文脈の記憶量）をゼロに戻します。長い会話が続いてコストが気になったときに使います。

## 3.6 よく使う便利コマンド一覧

Claude Code の起動後に使える「スラッシュコマンド（`/` から始まるコマンド）」の主なものをまとめました。

| コマンド | 説明 |
|---------|------|
| `/help` | ヘルプを表示する |
| `/cost` | 現在のセッションコストを表示 |
| `/clear` | 会話履歴をリセット |
| `/exit` | Claude Code を終了する |

---

## ここで詰まったら

| 症状 | 対処法 |
|------|--------|
| Claudeが止まらない | `Esc` を押す。それでも止まらなければ `Ctrl + C` |
| 間違ったファイルを変更された | `git restore .` で元に戻す |
| 費用が思ったより高い | 作業を小分けにして`/cost` で確認する習慣をつける |
| Claudeの回答が的外れ | 指示に「ただし〜すること」という制約を追加する |
