# 第8章：卒業演習 — 自分の繰り返し業務を自動化する

> **到達目標：** これまでの学びを使って、自分の実際の業務をClaude Codeで自動化できる
> **目安時間：** 90〜180分（業務の複雑さによる）
> **前の章：** [第7章 ←](./07-mcp-tools.md)

---

## 8.1 この演習のゴール

第1〜7章で学んだことを総動員して、**あなた自身の業務の中にある繰り返し作業を一つ選び、Claude Code で自動化する**のが卒業演習のゴールです。

この演習に「正解の答え」はありません。デザイナーの方なら「納品ファイルの整理」かもしれませんし、コンサルタントの方なら「毎月の報告書まとめ」かもしれません。あなたの業務に合わせて取り組んでください。

---

## 8.2 自動化する業務を選ぶ

### 良い「自動化候補」の条件

以下の条件に当てはまる業務が、Claude Code による自動化に向いています：

✅ **繰り返しが多い**：週次・月次など、同じ手順を定期的にこなしている  
✅ **ファイルを扱う**：テキスト・Excel・PDFなど、何らかのファイルが関わる  
✅ **手順が決まっている**：「まずAをして、次にBをして、最後にCを作る」という流れがある  
✅ **現在の手間が大きい**：毎回30分以上かかっている、または面倒でつい後回しにしている

### 自動化候補リスト（参考）

職種別によくある「繰り返し業務」の例：

**書類・文書系**
- 毎月の請求書作成（クライアントリストから一括作成）
- 週次・月次レポートのまとめ
- 提案書・見積もりのひな型カスタマイズ
- 議事録から翌週の宿題リスト作成

**情報整理系**
- 受信メールの分類・要約メモの作成
- 取材・インタビューメモから記事下書き生成
- 案件フォルダの進捗確認・未着手タスクの抽出
- リサーチ結果のまとめ・比較表作成

**管理・集計系**
- 作業時間ログから月次稼働時間の集計
- 収支記録から月次サマリー作成
- クライアント別の案件状況一覧更新

## 8.3 演習の進め方

### ステップ1：業務を言語化する（15分）

選んだ業務を、以下の形式で書き出してください。紙でもテキストファイルでも構いません。

```
業務名：

現在の手順（ステップ形式で）：
1. 〇〇をする
2. 〇〇をする
3. 〇〇をする

使うファイル・フォルダ：
- 入力：〇〇フォルダにある〇〇ファイル
- 出力：〇〇という形式で〇〇フォルダに保存

毎回かかっている時間：約〇〇分

どこが一番面倒か：
```

### ステップ2：Claude Code に相談する（15分）

書き出した内容をClaude Code に見せて、自動化の方針を相談します：

```
> 毎月末に行っている業務があります。
  現在の手順は以下の通りです：
  [ステップ1で書いた内容を貼り付ける]

  この業務をClaude Codeで自動化したいのですが、
  どのように進めれば良いか教えてください。
  まず、どのファイルを使うか確認させてください。
```

### ステップ3：小さく試す（30〜60分）

いきなり本番データで試さず、**サンプルファイルや少量のデータで動作確認**してから本番に適用するのがポイントです。

```
> まずサンプルとして、1ファイルだけで試してみましょう。
  test/ フォルダを作って、そこで動作確認してから
  実際のデータに適用します。
```

### ステップ4：CLAUDE.md に手順を記録する（15分）

うまくいった手順を CLAUDE.md に記録しておくと、次回以降の再現が楽になります：

```
> 今やった作業の手順を CLAUDE.md の「定型業務」セクションに追記して。
  次回から「月次レポートを作って」の一言で同じことができるようにしたい。
```

### ステップ5：振り返りと改善（15分）

自動化が完了したら、振り返ってみましょう：

- 何分かかっていた作業が何分になったか？
- 想定通りにいかなかった部分はどこか？
- CLAUDE.md や手順をさらに改善できるところはあるか？

## 8.4 詰まったときのサポートプロンプト

### 「どこから始めればいいかわからない」

```
> 私は〇〇（職種）です。
  毎月〇〇（業務）に〇〇時間かけています。
  Claude Code で自動化するとしたら、どこから始めればいいですか？
  まず確認したいことを3つ聞いてください。
```

### 「うまくいかない・エラーが出た」

```
> 〇〇という操作をしたら、以下のような結果になりました：
  [実際の状況や出力を貼り付ける]

  期待していた結果は〇〇でした。
  何が原因か教えてください。
```

### 「一度できたけど次にやったらうまくいかない」

```
> 前回うまくいった手順を試したところ、今回はうまくいきませんでした。
  CLAUDE.md に記録した手順を読んで、問題を特定してください。
```

## 8.5 卒業の基準

この演習の「卒業」の目安は次の通りです：

- **最低限**：自分の繰り返し業務を1つ選んで、Claude Code で試すことができた
- **標準**：手順が完成し、CLAUDE.md に記録して次回再現できる状態にした
- **発展**：自動化で時間が短縮され、「もう一つ試したい業務」が頭に浮かんでいる

完璧を目指す必要はありません。「一回試してみた」だけでも、Claude Code の実力と限界の両方が体感できているはずです。

## 8.6 卒業後の次の一歩

Claude Code を使い続けていくと、自然と「もっとこうしたい」というアイデアが出てきます。そのときに参考にしてほしい方向性を紹介します。

### ローカルファイルをもっと活用する

- 年間の資料を整理して「全ファイルのインデックス」を作る
- 自分専用のプロンプトテンプレート集をClaude Codeで管理する

### 自動化の対象を広げる

- 第7章の MCP を試して Notion や外部ツールと繋げる
- よく使う一連の指示をセットにして CLAUDE.md に登録する

### コミュニティで学び続ける

Claude Code の機能は日々進化しています。最新情報は：
- [Anthropic 公式ドキュメント](https://docs.anthropic.com/ja/docs/claude-code)
- [Claude Code の更新情報（英語）](https://github.com/anthropics/claude-code)
