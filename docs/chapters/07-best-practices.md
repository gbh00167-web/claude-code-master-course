## はじめに：AIは「魔法の杖」から「チームの新入社員」へ

Claude Codeは魔法ではありません。例えるなら、**「タイピングが人間離れして早く、世界中の一般的なプログラミング知識は持っているが、"あなたのプロジェクトの背景事情や社内ルール"は一切知らない、今日入社してきたばかりの極めて優秀な新入社員」**です。

新入社員に「よしなにやっといて」と仕事を丸投げすれば、当然ズレた成果物が上がってきます。この章では、優秀な新入社員（AI）を正しくマネジメントするための技術を学びます。

## 7.1 設定：CLAUDE.md による「事前教育」

### CLAUDE.md とは

プロジェクトのルートに置くと、Claude Code起動時に自動で読み込まれるルールブックです。

**AIに教え込むべきルールの例：**

```markdown
# プロジェクトのコーディング規約（AIへの絶対指示）

## 1. 技術スタックの制約
- フロントエンドは React 18（TypeScript）を使用し、常にFunctional Componentを使うこと
- スタイリングはTailwind CSS v3を使用し、インラインスタイルは絶対に使用しないこと

## 2. 命名規則とドメイン知識
- 本システムでは「顧客」を `Customer` ではなく `Client` と命名すること
- 「注文」は `Order` ではなく `Transaction` とすること

## 3. 禁止事項
- `any` 型の使用禁止
- console.log のコミット禁止
```

> 💡 **「禁止事項」を明確に書くことが重要** — ポジティブな指示（「〜してください」）だけでなく、「〜は絶対に使用しないこと」というネガティブな制約を明記する方がAIの逸脱を防ぐ効果が圧倒的に高い。

このプロジェクトの `CLAUDE.md` は既に設定済みです。テンプレートは `templates/CLAUDE.md.template` を参照してください。

## 7.2 管理：コストとトークンの最適化

### コストが雪だるま式に増える理由

LLMは過去の会話を毎回すべてAIサーバーに送信します。会話が長くなるほど、1回の指示でのコストが指数関数的に膨れ上がります。

```
1回目: 1,000文字送信
2回目: 1,000文字 + 今回の500文字 = 1,500文字送信
3回目: 1,500文字 + 今回の500文字 = 2,000文字送信...
```

### 3つの節約戦術

**戦術1：こまめな `/compact` と思い切った `/clear`**

```
/compact   ← 10〜15往復ごと。要約して圧縮（文脈は維持）
/clear     ← 別のタスクに切り替える時。完全リセット
```

**戦術2：段階的指示（丸投げ禁止）**

❌ **悪い（丸投げ）：**
```
> ECサイトのカート機能と決済システムを全部作って。
```

⭕️ **良い（段階的）：**
```
> まずカート機能のデータ構造の設計だけ考えてください。コードは書かないで。
```

**戦術3：CLAUDE.md でプロンプトを短縮**

毎回「TypeScriptを使って」と書かなくて済む分、1回あたりのトークンが節約できます。

## 7.3 セキュリティ：実務での注意点

### チーム開発でのベストプラクティス

1. **CLAUDE.md をGitで管理する** — チームメンバー全員が同じルールでAIを使えるようになる
2. **`.claudeignore` をGitで共有する** — 誰かが秘密情報を誤送信するリスクを防ぐ
3. **AIが書いたコードのPRには必ずレビューを入れる** — AIは「完璧なように見える間違ったコード」を書くことがある

### コードレビューの視点（AIが書いたコードを見る目）

- [ ] 指示していない余計な変更が混ざっていないか
- [ ] `console.log` が残っていないか
- [ ] ハードコードされた秘密情報が含まれていないか
- [ ] 存在しないAPIやライブラリを使っていないか（ハルシネーション）
- [ ] パフォーマンス上の問題はないか

## まとめ

- `CLAUDE.md` はAIへの「プロジェクト入門書」——最初に一度書けば毎回プロンプトに書く必要がなくなる
- コスト管理は「こまめな `/compact`」「段階的指示」「`.claudeignore`」の3本柱
- AIのコードは必ず人間がレビューする——最終責任はあなたにある
