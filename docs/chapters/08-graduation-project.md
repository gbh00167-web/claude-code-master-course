# 第8章：卒業演習 — ゼロからアプリを作る

> **学習目的：** 全章の学びを総動員し、ゼロから一つのアプリケーションを完成させる
> **学習成果：** Claude Code を使った AIDD のサイクル（設計→実装→テスト→デバッグ→リファクタリング）を独力で回せるようになる
> **目安時間：** 120〜180分
> **演習：** [`docs/exercises/ex-07.md`](../exercises/ex-07.md)（詳細な手順はこちら）

---

## 卒業演習のゴール：「マークダウン・メモ帳アプリ」の開発

### 要件

1. 画面の左半分にテキスト入力エリア（エディタ）、右半分にマークダウンのプレビューエリア
2. 入力したテキストがリアルタイムでマークダウンとしてプレビューされる
3. データはブラウザの `LocalStorage` に自動保存され、リロードしても消えない
4. 使用技術：Vite + React + TypeScript + Tailwind CSS

---

## 8.1 準備フェーズ：環境構築と安全網の設置

**この段階でGitとclaudigignoreを設定しないと、後で後悔します。**

```bash
# 空のフォルダを作成して移動
mkdir ~/ai-memo-app && cd ~/ai-memo-app

# Claude Code を起動
claude
```

```
> これからReact + TypeScript + Tailwind CSS で Vite を使ったメモ帳アプリの
  環境を構築します。必要なコマンドを教えてください。実行前に確認を取ること。
```

環境構築後の**必須作業：**

```bash
# 1. .claudeignore の作成
cat > .claudeignore << 'IGNORE'
node_modules/
dist/
.env
*.log
IGNORE

# 2. Gitの安全ネットを張る
git init
git add .
git commit -m "初期環境の構築"
```

## 8.2 開発フェーズ：AIとペアプログラミング

### Step 1：計画立案（コードを書かせない）

```
> 以下の要件を満たすマークダウン・メモ帳アプリを作ります。
  絶対にコードを書かずに、以下の計画書を提示してください：
  - コンポーネント分割の案
  - 必要なnpmライブラリ
  - 実装の順序

  【要件】
  1. 左半分にエディタ、右半分にリアルタイムプレビュー
  2. LocalStorage に自動保存
```

→ **人間がレビューして修正や合意をしてから次へ進む**

### Step 2：UIの実装（ロジック後回し）

```
> 計画は完璧です。ステップ1として、LocalStorage保存などのロジックは後回しにして、
  「左側に入力、右側にリアルタイムプレビュー」という画面のUI（見た目と基本的な状態管理）だけを実装してください。
  スタイリングはTailwind CSSでモダンでシンプルなデザインにしてください。
```

→ `npm run dev` でブラウザ確認 → 気になる点を言葉で伝えて微調整 → `git commit -m "UIの実装完了"`

### Step 3：ロジックの追加

```
> UIの確認ができました。ステップ2として、入力したメモのテキストがブラウザをリロードしても消えないように、
  LocalStorageへの自動保存と読み込みのロジックを追加してください。
  Reactの useEffect などを適切に使い、パフォーマンスに悪影響が出ないよう注意してください。
```

---

## 8.3 完成フェーズ：テスト・リファクタリング・ドキュメント

### Step 4：リファクタリング

```
> 機能は完璧に動いています。最後にプロジェクトの保守性を上げるために：
  1. LocalStorageの読み書きロジックを `useMemoData.ts` というカスタムフックに分離してください
  2. 各コンポーネントとカスタムフックにJSDoc形式のコメントを追加してください
  アプリの動作と見た目は絶対に壊さないでください。
```

### Step 5：ドキュメント作成と最終コミット

```
> プロジェクトのルートに README.md を作成してください。
  アプリの概要、技術スタック、起動手順（npm install と npm run dev）を
  マークダウン形式でまとめてください。
```

```
> 素晴らしい仕事です。現在のプロジェクトの変更内容を読み取り、
  Conventional Commitsの形式に従って適切なコミットメッセージを作成し、
  git commit まで実行してください。
```

## おわりに：AI駆動開発者（AI-Driven Developer）として

手を止めて、振り返ってみてください。

このアプリをゼロから作り上げるために、**あなたがキーボードで直接ソースコードをタイピングした回数は、一体何回だったでしょうか？**

おそらく、ターミナルでプロンプト（指示）を打つ時以外は、ほぼ1行もコードを書いていないはずです。

それでいて、あなたはアーキテクチャを決定し、ステップごとに品質をコントロールし、美しいコードベースとドキュメントを構築しました。

これこそが、**AI駆動開発（AIDD）の真髄**です。

> **「コードを書く人」から「AIを指揮してプロダクトを作る人」へ**
