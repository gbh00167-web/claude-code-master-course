## 2.1 インストール前に確認すること

Claude Codeをインストールする前に、**費用の仕組みを必ず理解しておいてください。** これを知らずに使い始めると、予想外の請求が届くことがあります。

### 料金の仕組み（重要）

Claude Code の利用料金には **2つの種類** があります。

| 種類 | 内容 | 費用 |
|------|------|------|
| Claude Code 本体 | ツール自体のサブスクリプション | 月額 $20〜（Max プラン） |
| API 使用料 | Claudeが思考・回答するたびに発生 | 使った分だけ従量課金 |

> **注意：** Claude Codeは `claude.ai` の有料プランとは別の契約です。すでにWeb版Claudeの有料プランに入っていても、Claude Codeは別途契約が必要な場合があります。最新の料金体系は[公式サイト](https://claude.ai/claude-code)で確認してください。

### コスト感の目安

普通に使った場合の月間コスト感：
- **軽い使い方**（1日に数回、短い作業）：月 $5〜15 程度
- **標準的な使い方**（毎日数回、中程度の作業）：月 $15〜50 程度
- **ヘビーな使い方**（1日中使う、大量のファイル処理）：月 $50〜 程度

詳細は[コスト・料金ガイド](../references/cost-guide.md)を参照してください。

## 2.2 インストール手順（Mac）

### ステップ1：ターミナルを開く

「ターミナル（Terminal）」はMacに最初から入っているアプリです。

1. 画面右上の虫眼鏡アイコン（Spotlight検索）をクリック
2. 「terminal」と入力してEnterを押す
3. 黒い（または白い）画面が開く——これがターミナルです

```
Last login: Fri Apr 25 10:00:00 on ttys000
yamaoka@Mac ~ %
```

`%` や `$` のマークが表示されていれば、コマンドを入力できる状態です。

### ステップ2：Claude Code をインストールする

ターミナルに以下を入力してEnterを押します（コピー＆ペーストで貼り付けてOKです）：

```bash
npm install -g @anthropic-ai/claude-code
```

> **「npm」って何ですか？**
> npm（エヌピーエム）は、プログラムをインストールするためのツールです。Macのターミナルから使える「アプリストア」のようなものです。
> `-g` はグローバルの略で、「どこのフォルダからでも使えるようにインストールする」という意味です。

インストールには1〜2分かかります。画面にいろんな文字が流れますが、最後に `done` や `added X packages` のような表示が出ればOKです。

### ステップ3：インストールを確認する

```bash
claude --version
```

`claude 1.x.x` のようにバージョン番号が表示されればインストール成功です。

> **ここで詰まったら：** `npm: command not found` と表示された場合
> Node.jsがインストールされていません。[nodejs.org](https://nodejs.org/ja) にアクセスし、「LTS（推奨版）」をダウンロードしてインストールしてください。完了後、ターミナルを一度閉じて開き直してから再度試してください。

## 2.3 はじめての起動

### 作業フォルダを決めて移動する

Claude Code は「今いるフォルダ」を起点として動作します。まず作業したいフォルダに移動してから起動するのが基本です。

```bash
# デスクトップの「仕事」フォルダに移動する例
cd ~/Desktop/仕事

# 移動できているか確認する（lsでフォルダ内のファイル一覧を表示）
ls
```

> **「cd」って何ですか？**
> `cd`（シーディー）は "change directory"（フォルダを変更する）の略です。
> `cd ~/Desktop/仕事` は「デスクトップの中にある『仕事』というフォルダに移動して」という命令です。
> `~`（チルダ）は「自分のホームフォルダ」を意味する特別な記号です。

### Claude Code を起動する

```bash
claude
```

はじめて起動すると、ブラウザが自動的に開きAnthropic（アンソロピック）のサイトへの認証を求められます。

1. ブラウザに Anthropic アカウントのログイン画面が表示される
2. アカウントを持っていなければ作成する（無料）
3. 「Authorize Claude Code」ボタンをクリックして許可する
4. ターミナルに戻ると `>` のプロンプトが表示される

```
✓ Claude Code 起動完了

> _
```

`>` のマークが表示されれば、Claudeに話しかけられる状態です。まず日本語で挨拶してみましょう：

```
> こんにちは。このフォルダに何がありますか？
```

Claudeが現在のフォルダの内容を読んで報告してくれます。

## 2.4 権限と安全の仕組みを理解する

Claude Code の重要な特徴のひとつは、**ファイルを変更する前に必ず確認を求める** という仕組みです。

### 「許可を求める」場面

Claudeがファイルを読むだけなら許可は求めません。しかし**ファイルを変更・作成・削除**しようとするとき、ターミナルにこのような確認が表示されます：

```
Claude wants to write to: 提案書_draft.txt
Do you want to allow this? [Y/n]
```

- `Y`（またはEnter）→ 許可する
- `n` → 拒否する

> **なぜ確認するのですか？**
> AIが「良かれと思って」重要なファイルを上書きしてしまうことを防ぐためです。「確認してから実行する」というのがClaude Codeのデフォルト（初期設定）の動作です。

### 安全ネットとしてのGit（バージョン管理ツール）

Claude Code を使い始める前に、作業フォルダをGitで管理することを強くお勧めします。Gitを使うと「変更を保存した時点に戻れる」ようになります。

```bash
# 作業フォルダをGit管理下に置く（初回のみ）
git init

# 現在の状態を保存する（セーブデータを作る）
git add .
git commit -m "作業前のセーブ"
```

> **Gitが難しそうで不安な方へ：**
> Gitは奥が深いツールですが、ここで覚えることは2つだけです。
> - `git add . && git commit -m "メモ"` → セーブする
> - `git restore .` → 最後のセーブ時点に戻す
>
> AIが変な変更をしてしまったら `git restore .` の一言で元に戻せます。これだけ知っていれば十分です。

## 2.5 機密情報の扱いと .claudeignore の設定

Claude Code はフォルダ内のファイルを自由に読めます。これは便利ですが、**Claudeに読ませてはいけないファイル** が存在する場合があります。

### 読ませてはいけないファイルの例

- パスワードや認証情報（`.env` ファイルなど）
- 顧客の個人情報（氏名・住所・連絡先が含まれるファイル）
- 契約で第三者に見せてはいけないと定められた資料

### .claudeignore で読み取り禁止にする

`.claudeignore`（クロードイグノア）というファイルを作ると、「このファイルはClaudeに読ませない」という設定ができます。

```bash
# .claudeignore ファイルを作る
claude
```

Claude Code を起動して、こう頼みましょう：

```
> このフォルダに .claudeignore ファイルを作って。
  含める内容：
  - .env（環境変数ファイル）
  - private/フォルダ以下のすべて
  - *_confidential.* （「_confidential」が付くファイル全部）
```

Claudeが自動的に適切な `.claudeignore` ファイルを作成してくれます。

> **「.env」って何ですか？**
> `.env`（ドットエンブ）はパスワードやAPIキー（認証コード）をまとめた設定ファイルです。このファイルはAIに渡さないのが原則です。

## 2.6 Claude Code 終了の方法

作業が終わったら、次のいずれかでClaude Codeを終了します：

```
/exit
```

または `Ctrl` + `C` を2回押します。

> **「セッション」って何ですか？**
> Claude Code を起動してから終了するまでの一連の会話のことを「セッション」と呼びます。セッションを終了すると、その会話の記憶はリセットされます（ただし CLAUDE.md に書いた設定は次回も引き継がれます）。

## ここで詰まったら

| 症状 | 対処法 |
|------|--------|
| `npm: command not found` | Node.jsをインストールしてターミナルを再起動 |
| `claude: command not found` | `npm install -g @anthropic-ai/claude-code` を再実行 |
| ブラウザが開かない | `claude --browser` オプションを試す |
| 日本語が入力できない | ターミナルの設定→テキストエンコーディングをUTF-8に変更 |

## 2.7 この章のまとめ

- Claude Code には本体サブスクリプション + API使用料の2種類のコストがある
- インストールは `npm install -g @anthropic-ai/claude-code` の一行
- 起動は `claude`、終了は `/exit`
- 作業前に `git commit` でセーブしておくと、いつでも元に戻せる
- `.claudeignore` で読ませたくないファイルを除外できる
