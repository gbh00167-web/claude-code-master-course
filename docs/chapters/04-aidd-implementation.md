# 第4章：AI駆動実装（AIDD） — コードを書く・直す

> **学習目的：** 機能実装・リファクタリング・コミットをClaudeに依頼し、意図通りにコードを編集させる
> **学習成果：** 「コーダー（AIが担当）」と「テックリード・レビュアー（あなた）」という新しい分業を実践できる
> **目安時間：** 90分
> **演習：** [`docs/exercises/ex-04.md`](../exercises/ex-04.md)

## 4.1 機能実装：小規模から中規模の変更

AI駆動開発（AIDD）の基本原則は、**「小さな変更から確実に行い、こまめにセーブ（コミット）する」**ことです。

### 最小単位の修正と「制約」の付与

❌ **悪いプロンプト（曖昧で制約がない）**
```
> ログイン画面に「新規登録」のリンクを足して。
```

⭕️ **良いプロンプト（具体的で制約が明確）**
```
> `src/components/LoginPage.tsx` を編集してください。
  「ログイン」ボタンのすぐ下に「アカウントをお持ちでない方はこちら」という
  リンク（遷移先: /register）を追加します。

  【制約事項】
  - 新しいnpmパッケージはインストールしないでください
  - スタイリングはTailwindの既存クラス（text-sm text-gray-500）を使ってください
  - Next.jsの標準の <Link> コンポーネントを使用してください
```

**ポイント：** ファイル名、追加する場所、仕様、デザインの希望、技術的な制約——これらを明確に伝えることで手戻りがなくなります。

### 複数ファイルにまたがる変更は計画を先に立てさせる

```
> 新しく「お問い合わせページ（/contact）」を追加したいです。
  実際のコードを書く前に、まず以下の2点だけ教えてください：
  1. 新規作成・編集が必要なファイルのリスト
  2. どのようなコンポーネント構成にするかの実装計画

  いきなりファイルの編集は行わないでください。
```

人間がレビューして合意できたら：
```
> 計画の構成はOKです。ただし、フォームの管理には useState ではなく
  react-hook-form を使用してください。この方針で実装を進めてください。
```

## 4.2 リファクタリング：コードの品質向上

### 巨大コンポーネントの分割

```
> `src/components/UserProfile.tsx` が肥大化して保守性が低くなっています。
  アプリの動作と見た目は一切変更せずに、以下の3つに分割してください：
  1. `UserHeader.tsx`（アバター画像と名前の表示部分）
  2. `UserBio.tsx`（自己紹介文とリンクの部分）
  3. `UserPosts.tsx`（過去の投稿一覧の部分）
  分割に伴うimport文のパス修正も漏れなく行ってください。
```

### Linterエラーを一括解消

```
> `npm run lint` と `npm run type-check` を自分で実行してエラーログを読み取り、
  プロジェクト全体のエラーがゼロになるまで必要なファイルをすべて修正してください。
  `any` 型になっている部分は処理内容から適切な型を推論して定義してください。
```

> 🚨 **AIがコードを壊した時の「Git」というタイムマシン**
> ```bash
> # AIが作業する前に必ずセーブ
> git add . && git commit -m "作業前"
>
> # AIがコードをぐちゃぐちゃにしたら
> git restore .  ← 一瞬で元に戻る
> ```

## 4.3 Git連携：コミットまで完結させる

### 人間による最終レビュー（査読）

**AI駆動開発における最重要の鉄則：AIが書いたコードを中身を見ずにコミットしてはいけない。**

```bash
# AIが変更した差分を人間の目で確認する
git diff
```

「意図した通りの修正だけが行われているか」「余計な変更が混ざっていないか」を確認してから `git add .` します。

### コミットメッセージの自動生成

```
> 現在 `git add` されている変更内容を読み取り、
  Conventional Commits（feat: 〇〇を追加, fix: 〇〇を修正）の形式で
  適切なコミットメッセージを作成して `git commit` を実行してください。
  変更の理由を含めた詳細な説明も日本語で記述してください。
```
