# 第6章：CLAUDE.md で「自分の仕事スタイル」を覚えさせる

> **到達目標：** 毎回同じ指示を打たなくて済む設定ファイルを作れる
> **目安時間：** 45分
> **前の章：** [第5章 ←](./05-writing-files.md)　**次の章：** [第7章 →](./07-mcp-tools.md)
> **演習：** [`docs/exercises/ex-06.md`](../exercises/ex-06.md)

## 6.1 「毎回同じことを言わなくていい」仕組み

第4章・第5章で学んだファイルの読み書きを実践していると、こんな不満が出てきます：

- 「毎回『ですます調で書いて』と言うのが面倒」
- 「私の名前とメールアドレスを毎回伝えなきゃいけない」
- 「業界用語の説明を毎回するのは無駄」

**CLAUDE.md**（クロード・エムディー）はこの問題を解決するファイルです。

### CLAUDE.md とは

CLAUDE.md は、「Claude Code を起動するたびに自動で読み込まれるメモ」です。

あなたの仕事のルール・好みのスタイル・よく使う情報をここに書いておくと、Claude Code は毎回それを「最初から知っている状態」でスタートします。

```
あなた → CLAUDE.md に「ですます調で書くこと」と書いておく
Claude Code → 起動するたびにそのルールを覚えた状態でスタートする
あなた → 毎回「ですます調で」と言わなくていい
```

---

## 6.2 CLAUDE.md を作る

### 作成方法

1. 作業フォルダに移動する
2. Claude Code を起動する
3. 「CLAUDE.md を作って」と頼む

```
> このフォルダに CLAUDE.md を作って。
  内容は後から教える。まずは空のファイルでいい。
```

または、テキストエディタ（メモ帳など）で直接 `CLAUDE.md` という名前のファイルを作ってもOKです。

### CLAUDE.md の基本構造

```markdown
# 私について

私は〇〇（職種）です。
事業内容：〇〇のサービスを提供しています。

# 文書の書き方

- 文体：ですます調
- 宛名があるときは「〇〇様」の形式
- 専門用語は使わず、平易な日本語で
- 箇条書きは読みやすく

# よく使う情報

- 自分の名前：山田太郎
- 屋号：山田デザイン事務所
- メールアドレス：yamada@example.com
- 所在地：東京都渋谷区〇〇（請求書・契約書で使う）

# やってはいけないこと

- 価格について勝手に書かない（必ず確認する）
- クライアント名を他の文書に使い回さない
```

## 6.3 職種別CLAUDE.mdテンプレート

### フリーランスデザイナー向け

```markdown
# 基本情報

私はフリーランスのグラフィックデザイナーです。
主な仕事：ロゴ、名刺、パンフレット、Webバナーの制作。

# 文書スタイル

- 文体：ですます調（クライアント向け文書）
- ヒアリングシートやブリーフは箇条書き
- 制作費の表記は「税込」で統一

# よく使う情報

- 屋号：〇〇デザイン
- 氏名：〇〇 〇〇
- 振込先：〇〇銀行 〇〇支店 普通 1234567
- 支払期限の表現：「請求書発行後30日以内」

# 制作物のルール

- ファイル名の形式：クライアント名_案件名_バージョン（例：yamada_logo_v01）
- 納品物のフォルダ構成：Draft/ と Final/ に分ける

# 書かないこと

- 見積金額は私が確認してから記入する（勝手に書かない）
- 著作権に関する記述は私がチェックしてから使う
```

### コンサルタント・士業向け

```markdown
# 基本情報

私は〇〇コンサルタントです。
対象クライアント：中小企業の経営者・個人事業主。
主な支援内容：〇〇の改善、〇〇の立ち上げ支援。

# 文書スタイル

- 提案書・報告書：ですます調、A4サイズ想定
- 箇条書きは「・」または「①②③」形式
- 専門用語は初出時に必ず括弧で説明
  例：KPI（重要業績評価指標）

# よく使う情報

- 氏名・資格：〇〇 〇〇（〇〇士）
- 事務所名：〇〇コンサルティング
- 電話：〇〇-〇〇〇〇-〇〇〇〇

# 資料作成の注意

- 現状分析→課題→提案→効果の順番で書く
- 数字は根拠を示す
- 結論を先に書き、詳細を後に続ける（PREP法）

# 機密情報

- クライアントの売上・経営情報は他の文書に書かない
- 法的な助言が必要な部分は「専門家にご相談ください」と明記する
```

### ライター・編集者向け

```markdown
# 基本情報

私はフリーライターです。
得意分野：〇〇、〇〇、〇〇。
主な媒体：Webメディア、企業オウンドメディア。

# 執筆スタイル

- 基本文体：です・ます調
- 文章のリズム：短文を基本とし、長文は2〜3文につき1回
- 見出し：疑問形または行動を促す形（例「〇〇するための3つの方法」）
- 冒頭：読者の悩みや状況を描写してから入る

# 原稿の単位

- Web記事：1500〜3000字
- コラム：800〜1200字
- ニュースリリース：400〜600字

# 取材・インタビューのルール

- 発言は「」で引用
- 敬称：〇〇さん（会社役員は〇〇氏）
- 事実確認が必要な部分には「要確認」タグをつける

# 書かないこと

- 取材先の未公開情報を文書に書かない
- SEOキーワードの無理な詰め込みはしない
```

## 6.4 CLAUDE.md を「育てる」考え方

CLAUDE.md は一度作ったら終わりではありません。使いながら少しずつ改善するものです。

### 「また同じことを言ってしまった」と気づいたら追加する

```
> 今教えたこと（「ファイル名には日付を入れること」）を
  CLAUDE.md に追加して。次回から毎回言わなくていいようにしたい。
```

Claudeが CLAUDE.md を更新してくれます。

### 定期的な見直しのタイミング

- **月1回**：先月使っていて「いつもこう指示してるな」と思ったことがあれば追加
- **問題が起きたとき**：「またこの間違いをした」と感じたら「やってはいけないこと」セクションに追記

### CLAUDE.md の限界を知っておく

CLAUDE.md に書いた内容は「ヒント」であり、Claudeが必ず守る「命令」ではありません。特に重要なルール（価格の取り扱いなど）は、都度明示的に指示するか、完成したものを必ず確認する習慣をつけてください。

---

## ここで詰まったら

| 症状 | 対処法 |
|------|--------|
| CLAUDE.md の設定が反映されない | ファイルが作業フォルダの直下にあるか確認。Claude Codeを再起動してみる |
| Claudeが CLAUDE.md を読んでいるか不明 | 「CLAUDE.mdに何が書いてありますか？」と聞いてみる |
| CLAUDE.md が長くなりすぎた | 優先度の低い内容を別ファイルに移し、CLAUDE.mdはシンプルに保つ |

---

## 6.5 この章のまとめ

- CLAUDE.md は「起動するたびに自動で読まれる設定メモ」
- よく使う情報・文書スタイル・禁止事項を書いておくと毎回の手間が減る
- 使いながら少しずつ育てていくもの
- 重要なルールは都度確認する習慣も維持する
