# 第2章：エージェント・ループと対話の作法

> **学習目的：** Claude Codeが「考え、行動する」プロセスを理解し、適切に制御する
> **学習成果：** エージェントの自律性をコントロールし、安全かつコスト効率よく作業を依頼できる
> **目安時間：** 45分
> **演習：** [`docs/exercises/ex-02.md`](../exercises/ex-02.md)

---

## はじめに：AIは「答える」から「動く」へ

料理に例えるなら、ブラウザ版のAIは「レシピを教えてくれる料理研究家」です。しかしClaude Codeは「実際にキッチンに立って料理をしてくれるシェフ」です。「ハンバーグを作って」と頼めば、勝手に冷蔵庫（フォルダ）を開けてひき肉を探し、玉ねぎを刻み、フライパンで焼いてくれます。

この自律的な行動を支えているのが**「エージェント・ループ」**という内部メカニズムです。

---

## 2.1 動作原理：エージェント・ループの構造

Claude Codeに指示を出すと、内部では以下の4つのステップが高速で循環（ループ）しています。

```
[あなたの指示]
       │
       ▼
 1. 観察 (Observation)    ── 関連ファイルを検索して読み込む
       │
       ▼
 2. 推論・計画 (Reasoning) ── 「このCSSクラスを変えれば良さそうだ」と計画
       │
       ▼
 3. 実行 (Action/Tool Use) ── 実際にファイルを書き換える
       │
       ▼
 4. 評価 (Evaluation)     ── linterやテストを実行し確認する
       │
       └─── (タスク未完了なら1に戻りループ)
       │
       ▼
[タスク完了・あなたへ報告]
```

**4. 評価がチャット型AIとの最大の違いです。** エラーが出れば自ら再推論し、別の解決策を試みます。

> 💡 **実務でのポイント：「Plan（計画）」フェーズで人間が介入する**
>
> ❌ **悪い指示：**「新しい決済機能を追加して。」
>
> ⭕️ **良い指示：**「新しい決済機能を追加したい。**コードを書く前に、修正するファイルと実装手順の計画をリストアップして教えて。**」

## 2.2 制御コマンド：基本の4大コマンド

| コマンド | 役割 | 使うタイミング |
|---------|------|--------------|
| `/ask <質問>` | 読み取り専用モード（ファイルを変更しない） | コードを理解したいだけの時 |
| `/compact` | 会話履歴を要約・圧縮してコスト削減 | 10〜15往復を超えた頃 |
| `/clear` | 会話履歴を完全リセット | 全く別のタスクに切り替える時 |
| `/exit` | Claude Codeを終了 | 作業終了時 |

### /ask の重要性

通常モードで「ログイン処理はどうなってる？」と聞くと、Claudeが気を利かせて「ついでにリファクタリングしておきました！」と、勝手にファイルを修正してしまうことがあります。`/ask` を使えばファイルが変更されることはありません。

## 2.3 安全と権限：ガードレールの設定

### Allow / Refuse の判断

Claudeが重要な操作をしようとした時、必ず確認メッセージが出ます。

```
Claude wants to execute: npm install axios
? Allow this execution? (y/n/A)
```

- **y（Yes）：** 今回だけ許可
- **n（No）：** 許可しない
- **A（Always allow）：** このセッション中は常に許可

> 🚨 見知らぬシェルスクリプトの実行や意図しないファイル削除は迷わず `n` を押してください。

### .claudeignore の設定

プロジェクトのルートに `.claudeignore` を作成して、Claudeに読ませたくないファイルを指定します。

**設定すべき理由：**
1. **セキュリティ：** `.env` ファイルに書かれたAPIキーがAIサーバーに送信されるリスクを防ぐ
2. **コスト：** `node_modules/` のような数万ファイルを誤読み込みしてトークンが爆増するのを防ぐ

```bash
# このプロジェクトの .claudeignore は設定済みです
cat .claudeignore
```

## まとめ

- エージェント・ループ（観察→推論→実行→評価）がCLIエージェントの核心
- **計画を先に出させる**ことで意図しない暴走を防げる
- `/ask` は安全なコード調査に必須のコマンド
- `.claudeignore` でセキュリティとコストの両方を守る
