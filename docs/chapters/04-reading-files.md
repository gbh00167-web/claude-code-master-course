> **到達目標：** ローカルのファイルをClaude Codeに読ませて、成果物を自動的に出力できる

## 4.1 基本操作：フォルダに移動してからClaude Codeを起動する

Claude Code はターミナルで「今いるフォルダ」を起点に動きます。まず作業したいフォルダに移動してから起動するのが基本です。

```bash
# 例：デスクトップの「業務」フォルダに移動
cd ~/Desktop/業務

# Claude Code を起動
claude
```

起動したら、まず現在地を確認してみましょう：

```
> 今どのフォルダにいますか？どんなファイルやフォルダがありますか？
```

Claudeがフォルダの内容を読んで報告してくれます。これが「ファイルを読む」の第一歩です。

## 4.2 単一ファイルを読ませる

### 議事録の要約（最も基本的なユースケース）

打ち合わせ後に議事録テキストファイルを用意して、Claude Code に読ませてみましょう。

```
> meeting-20250425.txt を読んで、次の3点をまとめて：
  1. 決まったこと（決定事項）
  2. 次回までに誰が何をするか（アクションリスト）
  3. 次回の議題候補
```

ポイントは **「何をどの形式でまとめるか」** を明示することです。「要約して」だけよりも、具体的なアウトプット形式を指定する方が使いやすい結果になります。

### ファイル名を直接指定する

Claude Code にファイルを読ませるには、ファイル名を会話の中でそのまま書くだけでOKです：

```
> proposal_draft.docx を読んで、改善点を3つ教えて
> 2024年度_売上レポート.xlsx の内容を要約して
> client-feedback.txt を読んで、ネガティブな意見だけ抜き出して
```

> **対応しているファイル形式について**
> Claude Code はテキストファイル（.txt, .md など）はそのまま読めます。
> Word（.docx）やExcel（.xlsx）は、変換が必要な場合があります。
> PDFは「pdfplumber」などのツールが入っていれば読める場合があります。
> まずは試してみて、読めないようなら「このファイルをテキストに変換してから読んで」と伝えましょう。

## 4.3 複数ファイルを一括処理する

### フォルダ内の全ファイルを横断して分析

```
> meeting-notes/ フォルダの中にある全ての .txt ファイルを読んで、
  今月中に完了すべきタスクの一覧を作って。
  担当者ごとにグループ化して、期限が近い順に並べて。
```

人間が10個のファイルを開いてメモしていた作業が、数分で完了します。

### ファイル名のパターンで絞り込む

```
> 「2025年4月」という名前が含まれるファイルを全部読んで、
  先月の業務実績をまとめて
```

```
> reports/ フォルダの中で、今週（2025年4月21日〜25日）に
  更新されたファイルを教えて
```

## 4.4 実践レシピ集

### レシピ1：メールの文章を整形・返信文を作る

テキストとして保存したメール文面や、メモアプリからコピーした文章を使って：

```
> customer-inquiry.txt を読んで、丁寧なビジネスメールの返信文を作って。
  - 宛名は「山田様」
  - 返信の冒頭は「お問い合わせいただきありがとうございます」で始める
  - 折り返し連絡する旨を伝える（具体的な日時はまだ書かない）
  作成した返信文は reply-draft.txt として保存して
```

### レシピ2：複数の見積もりや資料を比較する

```
> quotes/ フォルダに入っている3社の見積もりファイルを全部読んで、
  価格・納期・含まれる作業内容を表形式で比較してまとめて。
  オススメの選択肢にはその理由も一言添えて。
```

### レシピ3：長い資料の要点を素早く掴む

```
> contract-draft.txt を読んで、以下の点だけ教えて：
  - 契約期間
  - 支払い条件
  - 解約の条件（違約金など）
  - 注意すべき特記事項
  全文の要約は不要。上記4点だけ。
```

### レシピ4：フォルダ全体の棚卸し

月次・年次の資料整理に：

```
> 2024年/ フォルダの構造を調べて、どんなカテゴリの資料が
  どれくらい入っているか教えて。
  ファイルが多すぎるフォルダや、整理が必要そうな場所があれば指摘して。
```

### レシピ5：取材メモや音声書き起こしから記事下書き

```
> interview-notes-0425.txt を読んで、以下の構成でブログ記事の下書きを作って：
  - タイトル案（3つ）
  - リード文（150字程度）
  - 本文（3つの見出し、各300字程度）
  - まとめ（100字程度）
  専門用語は一般読者向けに解説を加えること。
  article-draft.md として保存して。
```

## 4.5 Claude が「読めない」と言ったときの対処法

ときどき「このファイルは読めません」「対応していない形式です」と言われることがあります。

### 対処法1：テキストに変換してから渡す

WordやExcelは、Mac標準の機能や別アプリでテキスト（.txt）に変換してから渡す方法があります。

```
> proposal.docx が読めない場合は、テキスト形式に変換する方法を教えて
```

### 対処法2：内容をコピーして「貼り付け」で渡す

```
> 以下の内容を元に、要約を作って：
  [ここに読ませたい文章をコピーして貼り付ける]
```

Claude Code でも通常のChatのように、テキストを直接貼り付けて渡すことができます。

### 対処法3：必要な情報だけをメモファイルに書き出す

Excelの数値をコピーして、シンプルなテキストファイルに貼り付けてから渡すのも有効です。

---

## ここで詰まったら

| 症状 | 対処法 |
|------|--------|
| 「ファイルが見つからない」と言われる | ファイル名のスペルや大文字小文字を確認。`ls` でファイル一覧を表示して確認 |
| Excelが読めない | テキスト（CSV形式）に変換してから渡す |
| 文字化けする | ファイルをUTF-8で保存し直す |
| 処理に時間がかかりすぎる | 対象ファイルを絞り込んでから再度指示する |
| 結果が長すぎて使いにくい | 「300字以内に」「箇条書きで5点だけ」などアウトプットを制約する |

---

## 4.6 この章のまとめ

- `cd` でフォルダに移動してから `claude` を起動するのが基本
- ファイル名を会話の中に書くだけでClaudeが読み込んでくれる
- 複数ファイルの横断処理もフォルダ名を指定するだけ
- 出力形式（箇条書き・表・字数制限など）を指定すると結果の精度が上がる
- 読めないファイルはテキストに変換してから渡す
