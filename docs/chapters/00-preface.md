## 0.1 はじめに：「コピペの限界」という壁

「AIがコードを書いてくれる時代」が到来し、私たちの開発スタイルは過去数年で劇的に変化しました。ChatGPTやWeb版のClaudeを使って、ちょっとした正規表現や関数のロジックを生成させたり、謎のエラーメッセージの原因を尋ねたりした経験は、今や多くのエンジニアが日常的に持っているでしょう。

しかし、ブラウザの画面越しに対話する今のスタイルに、どこか**もどかしさや限界**を感じていませんか？

少し複雑なタスクを依頼しようとすると、途端に以下のような「往復作業（ピンポン）」が発生します。

**コンテキスト伝達の苦労：**
「現在開発中のアプリはNext.jsのApp Routerを使っていて、UIにはTailwind CSSを使用しています。ディレクトリ構造はこうなっていて……」と、プロジェクトの前提条件を毎回プロンプトで説明し、膨大なコードを手動で貼り付ける必要がある。

**コピペと修正の手間：**
生成された見事なコードを自分のエディタにコピペしたものの、既存ファイルとのインポートパスがズレていたり、既存の変数名と整合性がとれずに結局手作業で微調整することになる。

**エラー対応の不毛な反復：**
いざ実行してエラーが出たら、ターミナルのログをコピーしてまたブラウザのAIに貼り付け、「このエラーが出ました、どうすればいいですか？」と尋ねる作業を繰り返す。

これらはすべて、**AIがあなたの開発環境から「切り離された外の世界」にいるために起こる摩擦**です。

---

## 0.2 なぜ「ブラウザ」を飛び出し「ターミナル」へ向かうのか？

**Claude Code** は、この摩擦を完全に無くします。AIをブラウザの檻から引きずり出し、あなたのターミナルの中に「超優秀なペアプログラミングの相手」として直接座らせるための画期的なツールです。

最大の特徴は、**「あなたのローカルファイルシステムに直接アクセスし、コマンドを実行できる」** という点に尽きます。

- **究極のコンテキスト共有：** 「このプロジェクトのルーティング設定はどうなってる？」と聞くだけで、Claude Codeは自らpackage.jsonを読んで技術スタックを理解し、ディレクトリツリーを走査して全体像を把握した上で回答してくれます。

- **自律的なファイル編集：** 「このバグを直して」と指示すれば、Claudeは原因を特定し、直接対象のファイルを書き換えてくれます。複数ファイルにまたがる複雑なリファクタリングも一息に完遂できます。

- **コマンド実行とループ：** ファイルの書き換え後、自らリンターやテストコマンドを実行し、もしエラーが出ればそのログを自分で読んで再修正する「自律的なループ（Agentic Loop）」を回すことができます。

## 0.3 本書の対象読者と最終ゴール

本書は以下のような方に向けて書かれています。

- チャット型AIの限界を感じている方
- GitHub Copilot からもう一歩先へ進みたい方
- 「コードを書く人」から「AIを指揮してプロダクトを作る人」へと変わりたい方

> ⚠️ **前提知識：** ターミナルの基本操作（cd, ls, grep）とGitの基礎知識、Node.jsの動作環境がお手元にあることを想定しています。

### 本書の最終ゴール

> **「Claude Codeを独力で使いこなし、実際の業務プロジェクトにおいて設計・実装・テスト・デバッグのサイクルをAIと共に高速に回せるようになること」**
