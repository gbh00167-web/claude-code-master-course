## 7.1 MCP とは何か

ここまでの章では「自分のMacにあるファイル」を読み書きすることを学びました。しかし実際の仕事では、NotionやGmailなど外部ツールのデータも扱います。

**MCP（Model Context Protocol）**（エムシーピー）は、Claude Code と外部ツールを繋ぐための「接続口」です。

> **MCPを電気のコンセントに例えると：**
> コンセントの形（規格）を統一することで、どんな電気製品でも使えるようになります。
> MCP も同様に、「この形式で繋げば Claude Code から使える」という規格を定めています。
> Notion・Gmail・Slack・Googleカレンダーなど、MCP に対応したツールを Claude Code から操作できるようになります。

---

## 7.2 MCP でできること（個人事業主向けの具体例）

| 連携するツール | できるようになること |
|--------------|------------------|
| **Notion** | NotionのページをClaude Codeから読み書き。議事録をNotionに直接保存 |
| **ブラウザ（Web検索）** | 「最新の〇〇の情報を調べて」とWebを検索させる |
| **ファイルシステム拡張** | Mac全体のファイルを横断して検索・整理 |
| **Slack** | Slackのメッセージを読んで要約・返信文を作成 |

## 7.3 MCP の設定方法

### 設定ファイルの場所

Claude Code の MCP 設定は、Mac上の設定ファイルに書きます：

```bash
# 設定ファイルの場所（Macの場合）
~/.claude/claude_desktop_config.json
```

### 基本的な設定の仕組み

設定ファイルに「どのMCPサーバーを使うか」を書くと、次回のClaude Code 起動時から使えるようになります。

```json
{
  "mcpServers": {
    "サーバー名": {
      "command": "起動コマンド",
      "args": ["引数"]
    }
  }
}
```

## 7.4 ブラウザ検索MCP の設定（最も簡単な例）

最初の一歩として、Claude Code にWeb検索をさせる設定をしてみましょう。

### Brave Search MCP の設定

```bash
# Claude Code を起動して設定を頼む
claude
```

```
> MCP を使って Web 検索ができるようにしたい。
  Brave Search MCP の設定方法を教えて、実際に設定してほしい。
```

Claude Code が設定手順を教えてくれます。大まかな流れは：

1. Brave Search の API キーを取得する（無料枠あり）
2. 設定ファイルに追記する
3. Claude Code を再起動する

設定が完了すると：

```
> 2025年4月に変わった確定申告のルールを調べて、要点をまとめて
```

のように、最新情報を調べながら回答してくれます。

## 7.5 Notion MCP の設定

Notionを使っている方には特におすすめです。

### Notion MCP の設定手順

**ステップ1：Notion インテグレーションを作成する**

1. [Notion Developers](https://www.notion.so/my-integrations) にアクセス
2. 「New integration」をクリック
3. 名前を入力（例：「Claude Code」）
4. 対象のワークスペースを選択して「Submit」
5. 表示された「Internal Integration Token」をコピーする

**ステップ2：Claude Code で設定する**

```
> Notion MCP を設定したい。
  Internal Integration Token は「ntn_xxxxxxxxxxxx」です。
  設定ファイルを更新して、次回起動から使えるようにして。
```

**ステップ3：使いたいNotionページの権限を付与する**

1. Notionで対象のページを開く
2. 右上の「…」メニュー → 「Connections」
3. 作成したインテグレーションを追加する

### Notion MCP を使った例

設定後：

```
> 「議事録」データベースの直近5件を読んで、アクションリストを作って

> 今日の打ち合わせメモを「議事録」データベースに新しいページとして追加して：
  タイトル：〇〇社との打ち合わせ（2025/04/25）
  参加者：山田、佐藤
  決定事項：〇〇を来月末までに提出する
```

## 7.6 MCPを使い始める前に知っておくこと

### MCPはやや上級の機能

MCP は Claude Code の中で最も設定が複雑な部分です。設定中にうまくいかないことも多いので、「うまく動かなくても当然」という心持ちで試してみてください。

### よくあるトラブルと対処法

| 症状 | 原因 | 対処法 |
|------|------|--------|
| 「MCP サーバーに接続できない」 | 設定ファイルの書き方が間違っている | Claudeに設定ファイルを確認してもらう |
| 認証エラーが出る | API キーや Token が間違っている | 再度コピーして確認する |
| 使えない機能がある | 権限（パーミッション）の設定不足 | Notionなら対象ページのConnection設定を確認 |

### トラブル時の頼み方

```
> MCP の設定でエラーが出ています。
  エラーメッセージは：[エラー文をここに貼り付け]
  設定ファイルの内容を確認して、問題を教えて。
```

---

## 7.7 「MCPなしでも十分」という選択肢

MCP が複雑すぎると感じたら、設定しなくても問題ありません。

第4章・第5章で学んだ「ローカルファイルの読み書き」だけでも、多くの業務自動化ができます。MCPはあくまで「さらに広げたい人向けの拡張機能」です。

**MCP なしでできること**の範囲で十分価値があるかどうかを、まず第4・5・6章の技術で判断してみてください。

---

## 7.8 この章のまとめ

- MCP は Claude Code と外部ツールを繋ぐ「接続口」
- Notion・Brave Search・Slack などと連携できる
- 設定はやや複雑なため、うまくいかなくても焦らず進める
- MCP は「拡張機能」——なくても十分に使える
