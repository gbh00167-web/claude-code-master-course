## 🏋️ 演習1-A：環境セットアップの確認

まず環境確認スクリプトを実行してください。

```bash
bash scripts/check-env.sh
```

問題があれば Claude に聞いてみましょう：

```
Node.js のインストールで EACCES というエラーが出ました。
Mac でどう解決すればいいか、初心者向けに教えてください。
```

---

## 🏋️ 演習1-B：インストールと初回起動

```bash
# Claude Code をインストール
npm install -g @anthropic-ai/claude-code

# バージョン確認
claude --version

# このディレクトリで起動
claude
```

起動したら、最初の一言を打ってみましょう：

```
> このプロジェクトの概要と主要なディレクトリ構成を教えて。
```

---

## 🏋️ 演習1-C：コスト確認の習慣をつける

上の質問を1〜2回した後、コストを確認してみましょう：

```
> /stats
```

📝 **確認ポイント**
- `in` と `out` のトークン数はいくつでしたか？
- 推定コストはいくらでしたか？
- 「この作業で約○円かかった」という感覚を掴みましょう

---

## 🏋️ 演習1-D（応用）：自分のプロジェクトを読み込ませる

手元に開発中のプロジェクトがある場合は、そのディレクトリで Claude Code を起動して試してみましょう。

```bash
cd ~/your-project
claude

> このプロジェクトのディレクトリ構造を教えて。技術スタックと、
  処理のエントリポイント（入り口となるファイル）を特定してください。
```

## ❓ よくある詰まりポイント

**Q: ブラウザが自動で開かない**
```
ターミナルに表示される https://... のURLを手動でコピーしてブラウザに貼り付けてください。
```

**Q: `EACCES` エラーでインストールできない（Mac）**
```
sudo をつけずに、nvm（Node Version Manager）を使ってNode.jsを入れ直すのが正解です。
Claude に「Mac で Node.js を nvm を使ってインストールする方法を教えて」と聞いてみましょう。
```
