# 第8章 演習（卒業演習）— ゼロからアプリを作る

> **目安時間：** 120〜180分
> **これがゴールです。全章の学びを総動員しましょう！**

---

## 🎯 卒業演習の目標

**「マークダウン・メモ帳アプリ」** を Claude Code と一緒にゼロから作る

### 要件
1. 画面の左半分にテキスト入力エリア（エディタ）、右半分にプレビューエリア
2. 入力したテキストはリアルタイムでマークダウンとしてプレビューされる
3. データはブラウザの LocalStorage に自動保存される（リロードしても消えない）
4. Vite + React + TypeScript + Tailwind CSS を使用

---

## 📋 演習の進め方（段階的に進むこと！）

### ステップ0：安全ネットの準備（必須）

```bash
# 空のフォルダを作って移動
mkdir ~/ai-memo-app
cd ~/ai-memo-app

# Claude Code を起動
claude
```

```
> これから React + TypeScript + Tailwind CSS で
  Vite を使ったメモ帳アプリの環境を構築します。
  必要なコマンドを教えてください。実行前に確認を取ること。
```

環境構築後、**必ず** 以下を実行：

```bash
# .claudeignore を作成
cat > .claudeignore << EOF
node_modules/
dist/
.env
*.log
EOF

# Gitの安全ネットを張る
git init
git add .
git commit -m "初期環境の構築"
```

### ステップ1：設計（コードを書かせない）

```
> 以下の要件を満たすマークダウン・メモ帳アプリを作ります。

【要件】
1. 左半分にテキスト入力エリア、右半分にマークダウンプレビュー
2. 入力したテキストはリアルタイムでプレビュー
3. LocalStorage に自動保存（リロード後も残る）

絶対にコードを書かずに、以下の計画書を提示してください：
- コンポーネント分割の案
- 必要な npm ライブラリ
- 実装順序（どの順番で何を作るか）
```

→ 計画が返ってきたら **人間の目で確認・修正**

---

### ステップ2：UIの実装（ロジック後回し）

```
> 計画は完璧です。
  ステップ1として、LocalStorage 保存などのロジックは後回しにして、
  「左側に入力、右側にリアルタイムプレビューされる」という
  画面の見た目（UI）と基本的な状態管理だけを実装してください。
  スタイリングは Tailwind CSS で、モダンでシンプルなデザインにしてください。
  足りないライブラリがあればインストールしてください。
```

→ `npm run dev` でブラウザ確認
