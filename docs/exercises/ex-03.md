# 第3章 演習 — 指示・確認・制御を練習する

> **対応テキスト：** `docs/chapters/03-agent-control.md`
> **目安時間：** 30分

---

## この演習のゴール

良い指示の出し方を体感し、途中で止める操作を確認する。

---

## 演習3-A：悪い指示 vs 良い指示を比較する

練習用フォルダで Claude Code を起動して、以下の2つを試してみてください。

**まず「曖昧な指示」を出す：**
```
> このフォルダについて教えて
```

**次に「具体的な指示」を出す：**
```
> このフォルダにある全てのファイルの名前と、
  それぞれのファイルサイズを一覧にしてください。
  Markdown の表形式で出力してください。
```

2つの回答の違いを観察してみてください。

---

## 演習3-B：制約を加えた指示を試す

```
> このフォルダにあるファイルについて説明してください。
  ただし：
  - 200字以内で
  - 箇条書きで3点だけ
  - 専門用語は使わずに
```

制約が結果に与える影響を確認してください。

---

## 演習3-C：途中停止を練習する

長い作業をして、意図的に途中で止めてみます。

```
> このフォルダの全ファイルを1つずつ詳しく説明してください。
  各ファイルについて200字以上説明してください。
```

説明が始まったら **`Esc` キーを押して** 止めてみましょう。

---

## 演習3-D：コストを確認する

```
/cost
```

このセッションでかかったコストを確認してください。

---

## 確認ポイント

- [ ] 曖昧な指示と具体的な指示で結果が変わることを確認した
- [ ] 制約（字数・形式など）で出力をコントロールできた
- [ ] `Esc` キーで作業を途中停止できた
- [ ] `/cost` でコストを確認できた
