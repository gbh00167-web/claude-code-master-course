# 第6章 演習 — MCPによる機能拡張

> **目安時間：** 30〜45分
> **注意：** MCP の接続には API キーや事前登録が必要なものがあります

---

## 🏋️ 演習6-A：現在の MCP 一覧を確認する

```bash
# ターミナルで実行（Claude Code の中ではなく）
claude mcp list
```

まだ何も入っていない場合は空のリストが返ります。

---

## 🏋️ 演習6-B：MCP の概念を理解する（/ask で質問）

```
> /ask MCP（Model Context Protocol）とは何ですか？
  「USB ポート」のアナロジーを使いながら、初心者にも分かるように説明してください。
  また、Web 版の Claude との違いも教えてください。
```

---

## 🏋️ 演習6-C（任意）：Brave Search MCP を試す

Brave Search の API キーを取得済みの場合：

```bash
# MCP サーバーを追加
claude mcp add brave-search npx -y @modelcontextprotocol/server-brave-search
```

```
> Brave Search を使って、Claude Code の最新バージョンに関する情報を調べてください。
```

# 第7章 演習 — 実務最適化とベストプラクティス

> **目安時間：** 30〜45分
> **ポイント：** 「魔法使い」から「マネージャー」へ

---

## 🏋️ 演習7-A：CLAUDE.md のルールを体感する

```
> CLAUDE.md を読んで、このプロジェクトのルールを把握してください。
  把握したルールを箇条書きで確認として教えてください。
```

---

## 🏋️ 演習7-B：ルール違反を意図的に試す

```
> docs/exercises/ex-01.md に新しいセクションを追加してください。
  スタイルは `style="color: red"` のインラインスタイルを使ってください。
```

⚠️ `CLAUDE.md` では「インラインスタイル禁止」とルール設定しています。
Claude がどう反応するか観察してみましょう。

---

## 🏋️ 演習7-C：自分の CLAUDE.md テンプレートを作る
