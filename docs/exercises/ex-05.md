# 第5章 演習 — ファイルを書かせる

> **対応テキスト：** `docs/chapters/05-writing-files.md`
> **目安時間：** 45分

---

## この演習のゴール

Claudeに新しいファイルを作らせ、既存ファイルを編集させる操作を体感する。

---

## 準備：作業前のセーブ

```bash
cd ~/Desktop/claude-practice
git add . && git commit -m "第5章演習前のセーブ"
claude
```

---

## 演習5-A：新しいファイルを作らせる

```
> 以下の内容でビジネスメールのテンプレートを作って、
  mail-template.txt として保存してください：

  件名：[件名を入れる]
  
  [宛名]様
  
  お世話になっております。[自分の名前]です。
  
  [本文を入れる]
  
  何卒よろしくお願いいたします。
  
  [自分の名前]
  [連絡先]
```

確認が出たら `Y` を押して保存させます。
`mail-template.txt` ができているか確認してください。

---

## 演習5-B：既存ファイルを編集させる

第4章で作った `meeting-notes.txt` の文体を変えてみます：

```
> meeting-notes.txt を読んで、内容は変えずに
  議事録を「社外の人が読んでも分かる丁寧な文体」に書き直してください。
  書き直した内容を meeting-notes-formal.txt として保存してください。
  （元の meeting-notes.txt は変更しないこと）
```

---

## 演習5-C：間違えたときのリカバリーを練習する

Claudeが意図と違う内容を作ったときの修正指示を練習します：

```
> mail-template.txt の件名部分を「【重要】」で始まるよう書き直してください。
  それ以外は変えないこと。
```

変更後に `git restore .` で元に戻してみましょう：

```bash
git restore .
```

元の状態に戻ったことを確認してください。

---

- [ ] 「元ファイルは変えないこと」という制約が効いていた
