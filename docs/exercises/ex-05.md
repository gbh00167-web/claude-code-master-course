# 第5章 演習 — デバッグとテスト

> **目安時間：** 30〜45分
> **ポイント：** エラーを「恐怖の対象」から「AIを動かす武器」に変える

---

## 🏋️ 演習5-A：わざとエラーを出してAIに修正させる

まず意図的にエラーを含む簡単なスクリプトを作成します：

```
> scripts/ フォルダに `sample-with-bug.ts` というファイルを作成してください。
  内容：文字列の配列を受け取り、各要素を大文字に変換して返す関数。
  ただし、わざと TypeScript のエラーが出るように `any` 型を使った
  バグ入りのコードで書いてください。
```

次に、そのバグを自動修正させます：

```
> scripts/sample-with-bug.ts に TypeScript のエラーがあります。
  ファイルを読んでエラーを特定し、適切な型定義に修正してください。
  修正後にエラーがないことを確認してください。
```

---

## 🏋️ 演習5-B：デバッグ用コードの挿入と削除

```
> scripts/sample-with-bug.ts の関数が正しく動いているか確認したい。
  関数の入り口と出口に console.log を仕込んで処理を可視化してください。
  確認が終わったら console.log をすべて削除してください。
```

---

## 🏋️ 演習5-C（応用）：テストコードの自動生成

```
> scripts/sample-with-bug.ts の関数に対してユニットテストを書いてください。
  テストファイルは scripts/sample-with-bug.test.ts として新規作成します。
  正常系・異常系・エッジケース（空配列、nullが含まれる場合など）を
  網羅してください。
```
