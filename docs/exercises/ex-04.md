# 第4章 演習 — ファイルを読ませてみる

> **対応テキスト：** `docs/chapters/04-reading-files.md`
> **目安時間：** 45分

---

## この演習のゴール

実際のファイルをClaude Codeに読ませて、成果物を作れるようになる。

---

## 準備：練習用ファイルを作る

まず、練習用のテキストファイルを作ります。
テキストエディタ（メモ帳、VSCode など）を開いて、以下の内容を `meeting-notes.txt` として保存してください：

```
会議日時：2025年4月25日（金）14:00〜15:00
参加者：山田、佐藤、田中

議題1：新規サービスの企画
・田中さんからSNS連携機能の提案あり
・山田が5月10日までに市場調査を行う
・次回会議で結果を共有する

議題2：今月の請求書処理
・佐藤が4月30日までに全クライアント分を送付する
・未回収の請求書が2件あり（Aさん、Bさん）
・佐藤がフォローアップメールを今週中に送る

次回会議：5月9日（金）14:00〜
```

---

## 演習4-A：ファイルを読ませて要約する

練習用フォルダに `meeting-notes.txt` を置いて、Claude Code を起動します。

```bash
cd ~/Desktop/claude-practice
claude
```

```
> meeting-notes.txt を読んで、以下の形式でまとめてください：

  ## 決定事項
  （箇条書き）

  ## アクションリスト
  担当者名：タスク（期限）の形式で

  ## 次回会議
```

---

## 演習4-B：出力形式を変えてみる

同じファイルを使って、今度は違う形式で出力させてみます：

```
> meeting-notes.txt を読んで、アクションリストだけを
  Slack に貼り付けられる形式（絵文字を使って見やすく）で作ってください。
```

出力が変わることを確認してください。

---

## 演習4-C（応用）：自分の実際のファイルで試す

手元にある実際の業務ファイルを使って試してみてください（機密情報が含まれる場合は慎重に）。

```
> 〇〇.txt を読んで、[あなたの欲しい出力を指定] してください。
```

---

## 確認ポイント

- [ ] `meeting-notes.txt` をClaudeに読ませることができた
- [ ] 指定した形式で出力が返ってきた
- [ ] 出力形式を変えると結果が変わることを確認した
- [ ] （応用）自分のファイルでも試してみた
