# 第4章 演習 — AI駆動実装（AIDD）

> **目安時間：** 45〜60分
> **重要：** 演習前に必ず `git add . && git commit -m "演習4前のセーブ"` を実行！

---

## 🏋️ 演習4-A：計画立案 → 人間レビュー → 実装のサイクル

以下のプロンプトを **そのまま** 使ってみましょう：

```
> templates/ フォルダに新しく `gitignore.template` というファイルを追加したい。
  実際に作成する前に、まずどんな内容を入れるべきか計画を教えて。
  ファイルの作成は計画を確認してから行うこと。
```

計画が返ってきたら：
- 「この内容で作ってください」と許可する
- または「〇〇の部分は不要です、△△を追加してください」と修正を指示する

---

## 🏋️ 演習4-B：制約付きの修正指示

```
> docs/references/commands.md を編集してください。
  ファイルの末尾に「## 🎓 練習メモ」というセクションを追加し、
  受講者が自由にメモを書けるよう、空白のスペースを確保してください。
  
  【制約】
  - 既存の内容は一切変更しないこと
  - 追加するのは末尾の新しいセクションのみ
  - 見出しのレベルは他のセクションと統一すること
```

---

## 🏋️ 演習4-C：コミットメッセージの自動生成

演習4-A か 4-B の変更を `git add` した後：

```
> 現在 git add されている変更内容を読み取り、
  Conventional Commits 形式（docs: 〇〇 など）で
  適切なコミットメッセージを作成して git commit を実行して。
```
